The Stroke Lifecycle & Persistence

Distinguishing between immediate feedback, application view, and canonical storage.

1. Visual Lifecycle (Front-end)
Ink API Trail (Active)
Immediate raw feedback via OS. Vanishes on pen-up.
App View (Static Layer)
Smoothed spline rendered locally in the React Canvas. Committed on pen-up.
Synced State (Yjs)
Broadcast to all open views. The "working memory" of the desk.
2. Persistence Lifecycle (Back-end)
Local-First DB (SQLite/IndexedDB)
Immediate local disk write. The "safety net" for offline work.
↓ Sync Bus ↓
Remote DB (Elements Table)
Canonical storage with spatial indexing (Bounding Boxes). Source for PDF export.

Optimization: "View-Port Only" Sync

To support notebooks with up to 50 pages (MVP constraint), the App View only renders the current page, while the Sync Bus maintains the full notebook state in the Local-First DB background.

Why Three States?

  • Ink API: Solves lag.
  • App View: Solves aesthetics (smoothing).
  • DB Layer: Solves reliability (offline/sync).
