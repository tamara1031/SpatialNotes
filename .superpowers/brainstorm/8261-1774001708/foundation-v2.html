Foundation: Custom Engine vs. Integrated SDK

SpatialNotes needs to balance "GoodNotes-like" responsiveness with development efficiency.

A

Custom Ink API Engine

Directly implement Web Ink API (DelegatedInkTrail) on an absolute-size HTML Canvas.

  • Pro: Full control over "GoodNotes" look & feel (pen physics, erasers).
  • Pro: Optimized for fixed physical sizes (A4/B5) from the ground up.
  • Con: Significant development effort for stroke smoothing, pressure sensitivity, and rendering.
B

tldraw SDK Integration

Embed the tldraw core as the rendering engine while wrapping it in SpatialNotes' physical-page UI.

  • Pro: Industry-leading Ink API implementation and performance (120Hz support).
  • Pro: Battle-tested collaboration (Yjs) and eraser logic.
  • Con: License considerations (tldraw v2 license is complex for commercial use).
C

Excalidraw SDK Integration

Leverage the Excalidraw library for the canvas.

  • Pro: Excellent "hand-drawn" aesthetic; very light and fast.
  • Con: More focused on "whiteboard" style than "notebook" style. Harder to adapt to pixel-perfect A4 printing.
D

diagrams.net (Draw.io) SDK

Integrate the industry standard for structured diagramming.

  • Pro: Unmatched for professional diagrams (UML, flowcharts, AWS/Azure icons).
  • Pro: Apache License 2.0 (extremely friendly for monetization).
  • Con: DOM/SVG-based (mxGraph) — not optimized for low-latency handwriting. The "feel" of a stylus will be significantly worse than GoodNotes.

Strategic Comparison

Inking Focus (A/B/C)

  • Feels like a pen on paper.
  • Priority on handwriting speed.
  • Perfect for students & creative notes.

Diagramming Focus (D)

  • Feels like a documentation tool.
  • Priority on structured layout.
  • Better for software engineers.
